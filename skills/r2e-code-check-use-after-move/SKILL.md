## 适用场景

C++文件(h/hpp/cpp)

## 判定逻辑

- **use-after-move**: 对已由 `std::move` 移出、或经移动构造/移动赋值**源对象已被掏空**的实体, 仍**读取/写入**其状态(标准库对象多为「有效但未指定状态」, 再使用有未定义或意外行为), 但部分类型承诺移后可安全销毁或赋值(不算问题).
- 在**已移动的对象**上调用仍假定旧值的 API 等应报告.

## 指令

扫描用户指定位置的代码文件, 检查是否存在在移动后仍使用源对象(除销毁或重赋值等安全用法)的风险, 并给出修复建议(在移动后只用结果对象、用 `std::optional` 表达置空、或重排使用顺序等)
