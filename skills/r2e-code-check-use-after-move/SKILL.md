## 指令(做什么)

- 扫描用户指定位置的代码文件。
- 检查是否存在对象在移动后仍被继续使用（除销毁或重赋值等安全用法）。
- 给出修复建议（仅使用结果对象、使用 `std::optional` 表达置空、重排使用顺序等）。

## 约束(怎么做)

- [范围] C++ 文件（h/hpp/cpp）
- [命中] 对已由 `std::move` 移出、或经移动构造/移动赋值后源对象已被掏空的实体，仍读取或写入其状态。
- [命中] 在已移动对象上继续调用仍假定旧值的 API。
- [排除] 仅销毁或重赋值等移后安全用法不标记。
