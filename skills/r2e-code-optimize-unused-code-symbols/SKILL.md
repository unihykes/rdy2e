## 指令(做什么)

- 扫描用户指定位置的代码文件。
- 检查是否存在从未被调用或引用的代码符号（变量、成员变量、全局变量、宏、函数、类等）。
- 给出修复建议（删除无用定义、收敛作用域、补充 `[[maybe_unused]]`、或在确需保留时增加意图注释）。

## 约束(怎么做)

- [范围] C++ 文件（h/hpp/cpp）
- [命中] 在可静态判定范围内，声明或定义后从未被有效引用/调用的符号（变量、成员变量、全局变量、宏、函数、类/结构体、别名等）。
- [命中] 同一翻译单元内可直接确认“仅定义无使用”的高置信度符号。
- [排除] `[[maybe_unused]]`、条件编译保留、导出 API、接口占位、序列化/反射/插件框架约定入口等显式保留语义不标记。
- [排除] 模板实例化、反射、注册宏、链接期引用等不易静态确认场景降级为“可能未使用”，不作确定性命中。
