## 指令(做什么)

- 扫描用户指定位置的代码文件。
- 检查是否存在不必要的临时对象（链式构造中间态、先构造后条件使用、返回值中间态等）。
- 给出修复建议（原位构造、延迟构造、删除冗余中间变量、将格式化放入真实使用分支）。

## 约束(怎么做)

- [范围] C++ 文件（h/hpp/cpp）
- [命中] 可静态判定的“先构造临时对象再立即转移/丢弃”的冗余中间态。
- [命中] 热路径上提前格式化字符串但仅在少数分支使用（如仅在抛异常分支使用的错误信息）。
- [命中] 返回值中存在仅做中转且无额外语义的中间变量。
- [排除] 中间变量承担可读性、调试断点或语义命名价值时不标记。
- [排除] 调整后可能改变异常时机、副作用顺序或导致重复计算时不标记。
- [排除] 涉及跨函数大改或并发语义变化的方案不标记。
