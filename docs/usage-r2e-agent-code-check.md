# r2e-agent-code-check 使用说明

## 适用场景

`r2e-agent-code-check` 面向 C++ 代码中的高风险语义问题，适用于需要一次性执行多项原子检查并汇总结果的场景，而非全盘风格审查。

## 重点覆盖的问题类型

- **编译期难以或无法可靠诊断**：需要结合数据流、对象生命周期、调用关系推理，常规编译器报错无法完整覆盖。
- **依赖运行时条件且易偶发**：同一段代码在不同参数、分支或时序下可能“有时正常、有时崩溃/未定义行为”。
- **排查成本高**：问题复现不稳定、影响范围隐蔽、修复需要跨函数或跨模块分析。

## 明确不覆盖

- **纯风格问题**：仅代码风格不统一、可读性偏弱，但语义与生命周期通常仍正确的问题。
- **通用格式规范**：如命名风格、空格缩进、注释格式等非语义安全问题。

## 推荐使用方式

- 需要综合检查时：调用 `r2e-agent-code-check`，由其按顺序执行各原子技能并统一汇总。
- 只关注单项风险时：优先直接调用对应原子 skill，减少无关扫描与输出噪音。

## 输出预期

- 返回按技能类别组织的检查结果，包含结论、问题定位和修复建议。
- 无问题项可省略，聚焦有效告警，便于快速修复。
