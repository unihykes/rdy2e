# 代码检查 Agent 指南

## 文档定位

本文面向开发者、维护者与插件使用者，说明 `r2e-agent-code-check` 的使用边界、适用场景与结果解读方式。

## 适用场景

当你需要一次性排查 C++ 代码中的高风险语义问题（而不仅是风格问题）时，优先使用代码检查 Agent。

典型场景包括：

- 问题与生命周期、数据流、调用关系相关，编译器难以稳定覆盖。
- 故障受运行时参数、分支或时序影响，表现为偶发崩溃或未定义行为。
- 需要跨函数、跨模块排查，人工定位成本较高。

## 不适用场景

以下场景不建议使用代码检查 Agent：

- 仅做命名、注释、排版等纯风格整理。
- 仅需格式化或静态风格统一（如缩进、换行、排序）。

## 推荐使用方式

- **综合检查**：直接调用 `r2e-agent-code-check`，由其按既定顺序执行原子技能并汇总结果。
- **单项排查**：只关注某一类风险时，直接调用对应原子 skill，减少扫描范围和输出噪音。

## 输出结果解读

你将获得按技能类别组织的检查结果，通常包含：

- 风险结论（是否命中）
- 问题说明（触发原因与影响）
- 修复建议（可落地的调整方向）

若某一项未发现问题，结果中可省略该项，以便聚焦有效告警。
