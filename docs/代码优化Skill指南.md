# 代码优化 Skill 指南

## 文档定位

本文面向开发者、维护者与插件使用者，说明 `r2e-code-optimize` 的定位、使用方式与输出预期。

## 适用场景

当你希望清理冗余代码、降低维护噪音、提升可读性与可维护性时，优先使用代码优化 Skill。

当前重点覆盖：

- 未使用符号与无用代码识别（如变量、成员变量、全局变量、宏、函数、类等）。

## 不适用场景

以下问题不属于代码优化 Skill 的职责：

- 运行时风险或错误类问题（如空指针、越界、悬垂生命周期）。
- 仅格式化层面的风格调整。

上述场景请改用 `r2e-code-check` 或其他对应能力。

## 推荐使用方式

- **综合优化**：调用 `r2e-code-optimize`，由其编排优化类原子技能并统一汇总。
- **定向优化**：只关注某一优化点时，直接调用对应原子 skill。

## 输出结果解读

输出按技能类别组织，通常包含：

- 可优化点
- 影响说明（收益与潜在影响范围）
- 建议动作（删除、收敛作用域、保留声明等）

结果以高置信度项为主；低置信度项会标注“需人工确认”。
